# ビジョン — code-review

## ミッション

コードを「おまじない」から「理解できるもの」に変える。

## 問題意識

- コードを書けても「なぜこれが動くのか」を人に説明できないことが多い
- 解説記事はあるが、自分のコードへの答えではない
- コードレビューの観点を学ぶ機会が少ない

## ターゲット

| ユーザー | 使い方 |
|---------|-------|
| 個人開発者 | 自分のコードの理解を深める |
| コード学習者 | 写経したコードの意味を知る |
| 教育者 | 生徒のコードに丁寧な解説を付ける |
| チームリード | コードレビューの観点を共有する |

## 差別化

**「コード説明本」スタイルの解説**

一般的なAIチャットと違い、code-reviewは常に以下の章構成で解説する：
1. このコードは何をするのか（全体像）
2. 仕組みの解説（処理の流れ）
3. 関数・クラス・変数の関係（構造説明）
4. なぜこう書くのか（設計理由・代替案）
5. どこに影響するのか（依存・副作用）
6. よく使われる専門用語の解説

**3段階の解説レベル**

- 🧒 小学生向け：身近なたとえ（料理・ゲーム）、専門用語禁止
- 🧑‍🎓 中高生向け：基本用語OK、段階的な説明
- 👨‍💻 エンジニア向け：計算量・設計パターン・代替実装まで

## 成功の定義

- 週1回以上使われる
- 解説結果を誰かに共有したくなる
- 「このコード、なるほど！」という感想が生まれる
